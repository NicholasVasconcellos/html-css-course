## Explanation of Why Remove Center on X for the Grid:

  when we do Justify Items Center
  * Each element will fit to it's content, then add necessary margin on each side
  * Otherwise the default is stretch: making the content width = parent width
  * so if nothing specified the content size = cellWidth, cellHeight


  so by removing the justify items center, we let the image component have the default size of col size
  then we scale it t0 35% of the width. 

  if we didn't do this, then the calculation for the width wold be different, and the scling would have to be different
  here is how that would play out:

  Grid > img box > img

  img.width = imgbox.width

  imgbox.width = img.width (since it's not set , and grid justify center makes it fit to child)

  Circular dependency

  browser fall back interrups and sets

  imgBox.width = img.DefaultWidth (from the png file)

  then img.width  = 0.35 * imgBox.width (bout a third of the parent)

  which is different than: img.width = 0.35 * FullColumnWidth, which is what happens when we left the justify = stretch by default

## Transition Flicker on Page Load

Problem: using `transition: all` causes elements to animate/jump when the page loads (e.g., fonts loading cause reflow)

Fix 1: Be specific about what you transition
  transition: color 0.3s;
  Only transition the properties you actually need, not `all`

Fix 2: Preload class (when you need multiple transitions)
  Add a class to body that disables transitions, remove it after load

  CSS:
  .preload * { transition: none !important; }

  HTML:
  <body class="preload" onload="document.body.classList.remove('preload')">

  This blocks all transitions until the page finishes loading
